# 고양 07_영원한행복 지역대회 안내부_QR 출입 관리 시스템

핸드폰 카메라로 QR 코드를 스캔해서 출입 가능 여부를 즉시 판정하는 시스템입니다.
Vercel에 배포하면 여러 사람이 같은 링크로 접속해서 같은 명단을 공유합니다.

## 구조

```
qr-access-vercel/
├── public/
│   └── index.html      ← 화면 전체 (QR 스캔, 인원 관리, 엑셀 업로드, 로그)
├── api/
│   ├── people.js        ← 인원 목록 조회/추가/수정/삭제
│   ├── log.js            ← 출입 로그 기록/조회
│   └── verify.js         ← QR 코드 서버측 검증 (위변조 방지)
├── package.json
└── .gitignore
```

## 배포 방법 (GitHub + Vercel)

1. 이 폴더 전체를 GitHub 새 리포지토리에 업로드합니다.
2. https://vercel.com 에서 GitHub 계정으로 로그인 후 "Add New Project"로 이 리포지토리를 가져옵니다(Import).
3. 빌드 설정은 건드릴 필요 없이 그대로 Deploy를 누릅니다.
4. 배포가 끝나면 `Storage` 탭에서 **KV(Key-Value) Database**를 하나 생성하고, 이 프로젝트에 연결(Connect)합니다.
   - 연결하면 `KV_REST_API_URL`, `KV_REST_API_TOKEN` 같은 환경변수가 프로젝트에 자동으로 추가됩니다.
5. Deployments 탭에서 다시 한번 "Redeploy"를 눌러 환경변수를 반영합니다.
6. 발급된 `https://프로젝트이름.vercel.app` 링크를 안내 담당자에게 전달하면 끝입니다.

## 사용 흐름

1. 관리자(김규영)가 PC/핸드폰에서 같은 링크로 접속 → **엑셀 일괄 등록** 탭에서 명단 업로드, 또는 **인원 관리** 탭에서 개별 등록
2. 등록된 인원에게 QR 이미지(PNG)를 전달 (카카오톡, 문자 등)
3. 안내 담당자가 같은 링크를 핸드폰 사파리/크롬으로 열고 **QR 스캔** 탭에서 카메라로 스캔
4. 모든 기기가 동일한 서버 데이터를 보므로, 누가 등록하고 누가 스캔하든 결과가 일치합니다.

## 보안 참고

QR 코드 자체는 `이름-전화번호 뒷4자리` + 위변조 방지 서명값만 담고 있으며,
서버(`api/verify.js`)에서 서명을 검증하고 실제 출입 가능 여부를 판정합니다.
서명 비밀키는 `api/verify.js`와 `public/index.html` 안의 `SECRET` 상수입니다.
보안을 더 강화하고 싶다면 이 값을 Vercel 환경변수로 옮기는 것을 권장합니다.
